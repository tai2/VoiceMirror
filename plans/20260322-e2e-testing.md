# Plan: E2E Testing Facility with WebdriverIO

> Date: 2026-03-22

## Goal

Add an E2E testing facility using WebdriverIO + Appium that runs on iOS Simulator and Android emulator. The central challenge is that the app's entire behaviour is driven by PCM audio samples from the device microphone — which is not controllable in automated environments. The solution is a specialized `IAudioRecordingService` implementation (`E2EAudioRecordingService`) that, instead of reading from hardware, opens a WebSocket connection back to the test runner. The test runner can then push arbitrary PCM chunks into the app on demand, deterministically triggering voice onset, silence detection, and the full state machine.

---

## Architecture Overview

```
┌─────────────────────────────────────────┐
│            Test Runner Process          │
│                                         │
│  ┌──────────────────────────────────┐   │
│  │  WebdriverIO test spec           │   │
│  │  (e2e/specs/*.spec.ts)           │   │
│  │                                  │   │
│  │  const bridge = new             │   │
│  │    E2EAudioBridge(9876)          │   │
│  │  bridge.sendVoice(300)  ──────── │ ──┼──► WS server :9876
│  │  bridge.sendSilence(2000) ────── │ ──┼──►
│  └──────────────────────────────────┘   │
└─────────────────────────────────────────┘
                                               ▲ WebSocket
                                               │ (raw Float32Array binary frames)
┌─────────────────────────────────────────┐    │
│         iOS Simulator / Android Emu     │    │
│                                         │    │
│  E2EAudioRecordingService               │    │
│    └─ E2EAudioRecorder                  │    │
│         ws = new WebSocket(wsUrl) ──────┼────┘
│         onmessage → callback(chunk)     │
│                                         │
│  useVoiceMirror  ◄──── onAudioReady ───┤
│    state machine ticks on each chunk    │
└─────────────────────────────────────────┘
```

The test runner's `E2EAudioBridge` is a WebSocket **server**; the in-app `E2EAudioRecorder` is the **client**. This inversion means the test runner controls all timing without the app needing to open a listening port (which would require native permissions).

## Step 3: `E2EAudioRecordingService`

### 3-1. `src/services/E2EAudioRecordingService.ts`

```typescript
import { Platform } from 'react-native';
import type {
  IAudioRecordingService,
  IAudioRecorder,
  AudioChunkEvent,
} from './AudioRecordingService';
import type {
  AudioRecorderCallbackOptions,
  PermissionStatus,
  SessionOptions,
} from 'react-native-audio-api';

// On Android emulator, `localhost` points to the emulator itself.
// The host machine is reachable at 10.0.2.2.
const WS_HOST = Platform.OS === 'android' ? '10.0.2.2' : 'localhost';
const WS_PORT = 9876;

class E2EAudioRecorder implements IAudioRecorder {
  private ws: WebSocket | null = null;
  private callback: ((event: AudioChunkEvent) => void) | null = null;

  start(): void {
    const url = `ws://${WS_HOST}:${WS_PORT}`;
    this.ws = new WebSocket(url);
    this.ws.binaryType = 'arraybuffer';

    this.ws.onopen = () => {
      console.log('[E2EAudioRecorder] connected to test bridge');
    };

    this.ws.onmessage = (event: MessageEvent<ArrayBuffer>) => {
      if (!this.callback) return;
      const samples = new Float32Array(event.data);
      this.callback({ chunk: samples, numFrames: samples.length });
    };

    this.ws.onerror = (e) => {
      console.error('[E2EAudioRecorder] WebSocket error', e);
    };
  }

  stop(): void {
    this.ws?.close();
    this.ws = null;
  }

  onAudioReady(
    _config: AudioRecorderCallbackOptions,
    callback: (event: AudioChunkEvent) => void,
  ): void {
    this.callback = callback;
  }

  clearOnAudioReady(): void {
    this.callback = null;
  }
}

export class E2EAudioRecordingService implements IAudioRecordingService {
  requestRecordingPermissions(): Promise<PermissionStatus> {
    // In E2E mode we always grant permission immediately.
    return Promise.resolve('Granted' as PermissionStatus);
  }

  setAudioSessionOptions(_options: SessionOptions): void {
    // no-op
  }

  setAudioSessionActivity(_active: boolean): Promise<void> {
    return Promise.resolve();
  }

  createRecorder(): IAudioRecorder {
    return new E2EAudioRecorder();
  }
}
```

### 3-2. Wire into `App.tsx`

```tsx
import { E2EAudioRecordingService } from './src/services/E2EAudioRecordingService';

const isE2E = process.env.EXPO_PUBLIC_E2E === '1';

const realServices = {

  encoderService: new RealAudioEncoderService(),
  decoderService: new RealAudioDecoderService(),
  recordingsRepository: new RealRecordingsRepository(),
};

const e2eServices = {
  ...realServices,

};
```

Only `recordingService` is replaced. The encoder, decoder, and repository remain real so that the E2E build exercises the full recording pipeline.

## Step 5: Test Bridge Helper

The bridge runs inside the test runner process (Node.js). It owns the WebSocket server, generates PCM sample buffers at specified dB levels, and sends them to the app at the same cadence as a real microphone (~100 ms intervals = 4096 frames at 44100 Hz).

### `e2e/helpers/E2EAudioBridge.ts`

```typescript
import { WebSocketServer, WebSocket } from 'ws';
import { SAMPLE_RATE, VOICE_THRESHOLD_DB, SILENCE_THRESHOLD_DB } from '../../src/constants/audio';

const CHUNK_FRAMES = 4096; // must match BUFFER_LENGTH in useVoiceMirror.ts
const CHUNK_DURATION_MS = (CHUNK_FRAMES / SAMPLE_RATE) * 1000; // ~92.9 ms

// Convert a target dB RMS value to a peak amplitude for a sine wave.
// sine RMS = peak / sqrt(2)  →  peak = rms * sqrt(2)
function dbToAmplitude(targetDb: number): number {
  const rms = Math.pow(10, targetDb / 20);
  return rms * Math.sqrt(2);
}

function makeSineChunk(amplitude: number, frameIndex: number): Float32Array {
  const samples = new Float32Array(CHUNK_FRAMES);
  const freq = 440; // Hz — arbitrary, just needs to be non-zero
  for (let i = 0; i < CHUNK_FRAMES; i++) {
    samples[i] = amplitude * Math.sin((2 * Math.PI * freq * (frameIndex + i)) / SAMPLE_RATE);
  }
  return samples;
}

function makeNoiseChunk(amplitude: number): Float32Array {
  const samples = new Float32Array(CHUNK_FRAMES);
  for (let i = 0; i < CHUNK_FRAMES; i++) {
    samples[i] = amplitude * (Math.random() * 2 - 1);
  }
  return samples;
}

export class E2EAudioBridge {
  private readonly wss: WebSocketServer;
  private client: WebSocket | null = null;
  private frameIndex = 0;

  constructor(private readonly port: number = 9876) {
    this.wss = new WebSocketServer({ port });
    this.wss.on('connection', (ws) => {
      this.client = ws;
      ws.on('close', () => { this.client = null; });
    });
  }

  async waitForConnection(timeoutMs = 15_000): Promise<void> {
    const deadline = Date.now() + timeoutMs;
    while (!this.client) {
      if (Date.now() > deadline) throw new Error('E2EAudioBridge: timed out waiting for app connection');
      await new Promise((r) => setTimeout(r, 100));
    }
  }

  /** Send audio above VOICE_THRESHOLD_DB for `durationMs` milliseconds. */
  async sendVoice(durationMs: number, overrideDb?: number): Promise<void> {
    const db = overrideDb ?? VOICE_THRESHOLD_DB + 10; // -25 dB: clearly above threshold
    await this.sendChunks(durationMs, db);
  }

  /** Send audio below SILENCE_THRESHOLD_DB for `durationMs` milliseconds. */
  async sendSilence(durationMs: number, overrideDb?: number): Promise<void> {
    const db = overrideDb ?? SILENCE_THRESHOLD_DB - 10; // -55 dB: clearly below threshold
    await this.sendChunks(durationMs, db);
  }

  private async sendChunks(durationMs: number, targetDb: number): Promise<void> {
    const numChunks = Math.ceil(durationMs / CHUNK_DURATION_MS);
    const amplitude = dbToAmplitude(targetDb);

    for (let i = 0; i < numChunks; i++) {
      this.sendChunk(makeNoiseChunk(amplitude));
      this.frameIndex += CHUNK_FRAMES;
      await new Promise((r) => setTimeout(r, CHUNK_DURATION_MS));
    }
  }

  private sendChunk(samples: Float32Array): void {
    if (!this.client || this.client.readyState !== WebSocket.OPEN) {
      throw new Error('E2EAudioBridge: no client connected');
    }
    // Send as raw binary ArrayBuffer — matches `binaryType = 'arraybuffer'` in the recorder.
    this.client.send(Buffer.from(samples.buffer));
  }

  async close(): Promise<void> {
    return new Promise((resolve) => {
      this.wss.close(() => resolve());
    });
  }
}
```

## Step 7: Write E2E Tests

### `e2e/specs/voiceMirror.spec.ts`

```typescript
import { E2EAudioBridge } from '../helpers/E2EAudioBridge';
import {
  VOICE_ONSET_MS,
  MIN_RECORDING_MS,
  SILENCE_DURATION_MS,
} from '../../src/constants/audio';

describe('VoiceMirror — core loop', () => {
  let bridge: E2EAudioBridge;

  before(async () => {
    bridge = new E2EAudioBridge(9876);
    // App connects on startup. Wait before proceeding.
    await bridge.waitForConnection();
  });

  after(async () => {
    await bridge.close();
  });

  beforeEach(async () => {
    // Ensure app is in idle phase before each test.
    // Sending silence for long enough to settle any in-progress state.
    await bridge.sendSilence(SILENCE_DURATION_MS + 500);
    await $('~phase-idle').waitForDisplayed({ timeout: 10_000 });
  });

  it('shows idle phase on launch', async () => {
    await expect($('~phase-idle')).toBeDisplayed();
  });

  it('transitions to recording when voice is sustained', async () => {
    // Voice must be sustained for VOICE_ONSET_MS before recording starts.
    await bridge.sendVoice(VOICE_ONSET_MS + 200);
    await $('~phase-recording').waitForDisplayed({ timeout: 5_000 });
  });

  it('does not record on brief voice bursts shorter than onset threshold', async () => {
    await bridge.sendVoice(VOICE_ONSET_MS - 100); // just under threshold
    await bridge.sendSilence(500);
    await expect($('~phase-idle')).toBeDisplayed();
    await expect($('~phase-recording')).not.toBeDisplayed();
  });

  it('transitions from recording to playing after sustained silence', async () => {
    // Trigger recording.
    await bridge.sendVoice(VOICE_ONSET_MS + 200);
    await $('~phase-recording').waitForDisplayed({ timeout: 5_000 });

    // Keep recording long enough to pass MIN_RECORDING_MS.
    await bridge.sendVoice(MIN_RECORDING_MS + 200);

    // Now go silent long enough to stop recording.
    await bridge.sendSilence(SILENCE_DURATION_MS + 500);
    await $('~phase-playing').waitForDisplayed({ timeout: 10_000 });
  });

  it('returns to idle after playback completes', async () => {
    // Drive a full cycle.
    await bridge.sendVoice(VOICE_ONSET_MS + 200);
    await $('~phase-recording').waitForDisplayed({ timeout: 5_000 });
    await bridge.sendVoice(MIN_RECORDING_MS + 200);
    await bridge.sendSilence(SILENCE_DURATION_MS + 500);
    await $('~phase-playing').waitForDisplayed({ timeout: 10_000 });

    // Playback duration equals the recording duration (short).
    // After it ends, app returns to idle.
    await $('~phase-idle').waitForDisplayed({ timeout: 15_000 });
  });

  it('adds a recording to the list after the loop completes', async () => {
    await bridge.sendVoice(VOICE_ONSET_MS + 200);
    await $('~phase-recording').waitForDisplayed({ timeout: 5_000 });
    await bridge.sendVoice(MIN_RECORDING_MS + 200);
    await bridge.sendSilence(SILENCE_DURATION_MS + 500);
    await $('~phase-playing').waitForDisplayed({ timeout: 10_000 });
    await $('~phase-idle').waitForDisplayed({ timeout: 15_000 });

    // At least one recording should now appear in the list.
    const items = await $$('[testID^="play-recording-"]');
    await expect(items.length).toBeGreaterThanOrEqual(1);
  });
});

describe('VoiceMirror — pause / resume', () => {
  let bridge: E2EAudioBridge;

  before(async () => {
    bridge = new E2EAudioBridge(9876);
    await bridge.waitForConnection();
  });

  after(async () => { await bridge.close(); });

  it('pauses and resumes monitoring', async () => {
    await $('~phase-idle').waitForDisplayed({ timeout: 10_000 });

    await $('~toggle-pause-button').click();
    await $('~phase-paused').waitForDisplayed({ timeout: 5_000 });

    // Sending voice while paused must not trigger recording.
    await bridge.sendVoice(VOICE_ONSET_MS + 500);
    await expect($('~phase-recording')).not.toBeDisplayed();

    await $('~toggle-pause-button').click();
    await $('~phase-idle').waitForDisplayed({ timeout: 5_000 });
  });
});
```

## Design Notes

### Why WebSocket (app as client)?

The alternative — running an HTTP server inside the app — requires a native module. React Native's JS runtime has no `net.createServer`. Connecting outward via WebSocket uses only the standard `WebSocket` global, which React Native polyfills natively. Putting the server in the test runner also gives the test full control of when data is sent without race conditions.

### Why binary frames?

Each 4096-frame `Float32Array` is 16 KB. Sending it as a JSON array of numbers would be ~80 KB and add parse overhead. Using `binaryType = 'arraybuffer'` and raw `Buffer.from(samples.buffer)` keeps frame delivery fast and consistent with the ~93 ms cadence of a real microphone.

### Why keep encoder/decoder/repository as real implementations?

Swapping only the recording service tests the entire downstream pipeline (encoding to M4A, saving metadata, decoding for playback) under real conditions. Replacing these with stubs would undermine the purpose of E2E testing.

### Why `EXPO_PUBLIC_E2E` instead of a separate entry point?

A separate entry file (`App.e2e.tsx`) would need changes to `app.json` and adds maintenance overhead. `EXPO_PUBLIC_*` env vars are statically inlined by the Expo bundler — tree-shaking removes the unused branch — so there is no runtime cost and no extra file to maintain.

### Android host address

On Android emulator, `localhost` / `127.0.0.1` resolves to the emulator's own loopback, not the host machine. The host machine is reachable at `10.0.2.2`. The `E2EAudioRecordingService` detects `Platform.OS === 'android'` and uses the correct address automatically.

### PCM amplitude for threshold crossing

The state machine compares RMS dB against `VOICE_THRESHOLD_DB` (−35) and `SILENCE_THRESHOLD_DB` (−45). The bridge adds a 10 dB margin:
- Voice: target −25 dB (safely above −35)
- Silence: target −55 dB (safely below −45)

This avoids flakiness from floating-point rounding at the threshold boundary.
